<decisions_made>

- **AuthRepositoryImpl fix**: `refreshAccessToken()` now uses `withContext(Dispatchers.Main)` for AppAuth's `performTokenRequest()` call. AppAuth creates an Android `Handler` internally requiring a Looper — OkHttp background threads have no Looper, causing `RuntimeException` wrapped as `IOException`, misclassified as `NetworkError` ("No internet connection"). Fix posts the callback to main thread.
- **YouTubeRepositoryImpl fix**: `deletePlaylist()` replaced `runCatching` (which silently swallowed all errors) with targeted `try/catch(HttpException)` that only suppresses 404 (playlist already deleted). 401/403 now propagate to use case's catch blocks for correct error mapping.
</decisions_made>
